# 做题工具栏 UI 修复与统一 Spec

## Why
工具栏在实现新样式（线性图标、可拖动、弹出设置面板）的过程中，CSS 布局被破坏，界面出现样式混乱。需要通过截图比对恢复正确的界面外观，同时保留已实现的线性图标工具栏功能。

## What Changes
- 修复完形填空页面布局，恢复文章/选项两栏正常显示
- 修复真题精读页面布局，恢复文章/题目正常显示
- 确保线性图标工具栏在两种模式（完形填空/真题精读）中样式完全一致
- 修复拖拽跳位问题
- 修复弹出面板位置和自动关闭
- 统一两种模式的工具栏功能和交互

## Impact
- Affected specs: pastPapers, clozeExercise, pastPaperReader
- Affected code: css/pastPapers.css, js/clozeExercise.js, js/pastPaperReader.js, js/pastPapers.js

## ADDED Requirements

### Requirement: 工具栏线性图标样式
工具栏为横向一排圆角长条容器，内部依次平铺单色线条图标，无文字；图标顺序：撤回箭头、橡皮擦、铅笔书写、鼠标选择箭头、斜切橡皮擦、删除垃圾桶。当前激活的图标用白底方框突出。

#### Scenario: 完形填空工具栏显示
- **WHEN** 用户进入完形填空页面
- **THEN** 工具栏显示在文章区域下方，水平居中，6个图标正确排布

#### Scenario: 真题精读工具栏显示
- **WHEN** 用户进入真题精读页面
- **THEN** 工具栏显示在文章区域下方，水平居中，与完形填空样式完全一致

#### Scenario: 图标点击切换模式
- **WHEN** 用户点击笔图标
- **THEN** 当前模式切换为书写模式，笔图标高亮显示为白底方框
- **AND** 弹出颜色/粗细/透明度设置面板在工具栏正下方

#### Scenario: 设置面板自动关闭
- **WHEN** 用户在设置面板打开时点击面板外部任意区域
- **THEN** 设置面板自动关闭

### Requirement: 工具栏可自由拖动
工具栏容器支持鼠标拖拽移动位置。

#### Scenario: 第一次拖拽无跳位
- **WHEN** 用户首次点击工具栏并开始拖拽
- **THEN** 工具栏平滑切换到 fixed 定位，不发生位置跳跃

### Requirement: 鼠标光标对应模式
- 选择模式：默认箭头光标
- 书写模式：铅笔SVG图标光标
- 橡皮擦模式：橡皮擦SVG图标光标

### Requirement: 画布笔记跨页面保留
- 切换完形填空/真题精读/翻译练习模块时，画布笔记保留
- 同一模块内切换题目时，画布笔记保留
- 点击返回按钮时自动保存画布状态

## REMOVED Requirements
### Requirement: 旧版文字按钮工具栏
**Reason**: 替换为新的线性图标工具栏
**Migration**: 新工具栏与旧工具栏功能一一对应，但采用纯图标无文字样式
