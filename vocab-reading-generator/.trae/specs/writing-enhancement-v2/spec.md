# 考研真题模块功能增强V2 - 产品需求文档

## Overview
- **Summary**: 对作文指导和长难句模块进行全面增强，包括作文数据提取修复、优质范文扩充、针对性写作技巧、AI批改功能框架、书写模式圈画增强、橡皮擦增强、长难句数据扩充等
- **Purpose**: 提升作文学习和长难句分析的完整性和专业性
- **Target Users**: 考研英语学习者

## Goals
- 修复作文数据提取问题，正确处理表格、图表、图片等复杂内容
- 收集真实优质的参考范文，每篇作文提供多篇范文
- 针对每篇作文题目提供专业、针对性的写作技巧指导
- 添加作文答题区域，模拟真实考试环境
- 设计AI作文批改功能框架和不同类型作文的prompt提示词
- 大幅扩充长难句数据（目标：覆盖2000-2025年全部真题）
- 提升长难句解析质量，参考专业考研英语长难句解析标准
- 增强书写模式，支持圈画标注（颜色、粗细、形状、透明度）
- 增强橡皮擦功能，支持切换大小和形状

## Non-Goals (Out of Scope)
- 不实现AI批改的API调用（只完成前端框架和prompt设计）
- 不实现云端存储用户作文
- 不实现作文模板下载功能

## Background & Context
现有作文指导和长难句模块存在以下问题：
1. 作文题目提取逻辑无法处理复杂内容（表格、图表、图片）
2. 参考范文质量差，全部敷衍生成
3. 写作技巧千篇一律，没有针对性
4. 缺少用户答题区域
5. 缺少AI批改功能框架
6. 长难句数量不足（仅120句）
7. 长难句解析质量参差不齐
8. 书写模式功能单一，缺少圈画工具

## Functional Requirements

### FR-1: 作文数据提取修复
- 正确处理作文题目中的表格内容
- 正确处理作文题目中的图表描述
- 正确处理作文题目中的图片说明
- 对于复杂格式，提供降级显示方案

### FR-2: 优质参考范文扩充
- 从网络收集**真实**的考研英语作文范文
- 每篇作文题目提供**2-3篇**高质量参考范文
- 范文按难度或风格分类
- 标注范文来源和特点

### FR-3: 针对性写作技巧指导
- 真正阅读每篇作文题目
- 针对题目类型（图表作文、书信、应用文、图画作文等）提供专业技巧
- 包含：结构建议、开头结尾技巧、常用句型、注意事项
- 不同类型作文的技巧完全不同

### FR-4: 作文答题区域
- 添加类似考试的作文答题区域
- 支持多行文本输入
- 支持字数统计显示
- 支持草稿保存（localStorage）

### FR-5: AI作文批改功能框架
- 设计作文批改页面结构
- 设计不同类型作文的prompt提示词：
  - 图表作文prompt
  - 书信/应用文prompt
  - 图画作文prompt
  - 提纲作文prompt
- AI需能识别用户写的是哪篇作文
- 批改结果展示：分数、各维度评分、详细批注、改进建议
- 预留API接入接口

### FR-6: 长难句数据大幅扩充
- 从2000-2025年**全部**真题中提取长难句
- 数量目标：至少300句以上
- 按年份、试卷类型、来源篇目分类
- 确保每句都是真正的长难句（复杂语法结构）

### FR-7: 长难句解析质量提升
- 参考专业考研英语长难句解析标准
- 规范解析格式：
  - 主干提取：主语+谓语+宾语/表语
  - 从句分析：从句类型、功能、作用
  - 词汇注释：重点词汇、固定搭配
  - 结构图示：可选的树形结构
- 确保解析准确、专业

### FR-8: 书写模式圈画增强
- 在做题页面（完形填空、真题精读）添加画布层
- 支持多种画笔工具：
  - 颜色选择：预设颜色 + 自定义颜色
  - 粗细调节：细/中/粗/极粗
  - 形状选择：圆形、方形、荧光笔
  - 透明度调节：10%-100%
- 工具栏位置：页面顶部或侧边

### FR-9: 橡皮擦功能增强
- 支持橡皮擦大小调节：小/中/大/极大
- 支持橡皮擦形状：圆形、方形
- 支持橡皮擦模式切换（与画笔分离）

## Non-Functional Requirements
- **NFR-1**: 界面风格统一，与现有设计语言一致
- **NFR-2**: 响应式布局，适配不同屏幕宽度
- **NFR-3**: 画布操作流畅，无明显延迟
- **NFR-4**: 作文答题区域支持中英文输入

## Constraints
- **Technical**: 纯前端实现，画布使用Canvas API，AI批改预留接口
- **Business**: 作文范文和长难句数据需人工审核确保质量
- **Dependencies**: 依赖现有pastPapers模块架构

## Assumptions
- 作文真题数据中可以提取出题目内容（即使格式复杂）
- 用户需要手写作文输入功能
- AI批改API可以后续接入
- Canvas API可以满足圈画需求

## Acceptance Criteria

### AC-1: 作文数据正确显示
- **Given**: 用户选择一套真题进入作文指导
- **When**: 页面加载完成
- **Then**: 作文题目（包括表格、图表等）正确显示，无遗漏或格式错误
- **Verification**: `human-judgment`

### AC-2: 参考范文优质且多样
- **Given**: 用户展开参考范文
- **When**: 查看范文
- **Then**: 每篇作文有2-3篇高质量范文，来源真实，不是敷衍生成
- **Verification**: `human-judgment`

### AC-3: 写作技巧有针对性
- **Given**: 用户展开写作技巧
- **When**: 查看技巧
- **Then**: 技巧针对该篇作文题目撰写，不是通用模板
- **Verification**: `human-judgment`

### AC-4: 作文答题区域可用
- **Given**: 用户在作文指导页面
- **When**: 查看作文题目
- **Then**: 有答题区域可以输入自己的作文，有字数统计
- **Verification**: `human-judgment`

### AC-5: AI批改功能框架完整
- **Given**: 用户完成作文并点击提交
- **When**: 系统处理
- **Then**: 显示"AI批改功能待接入API"的提示，或展示批改结果框架（待API接入）
- **Verification**: `programmatic`

### AC-6: 长难句数量充足
- **Given**: 用户进入长难句分析模块
- **When**: 浏览长难句
- **Then**: 长难句总数 >= 300句，覆盖2000-2025年真题
- **Verification**: `programmatic`

### AC-7: 长难句解析质量高
- **Given**: 用户查看长难句解析
- **When**: 分析句子
- **Then**: 解析格式规范，分析专业准确
- **Verification**: `human-judgment`

### AC-8: 圈画工具丰富
- **Given**: 用户进入书写模式
- **When**: 使用圈画工具
- **Then**: 可以选择颜色、粗细、形状、透明度
- **Verification**: `human-judgment`

### AC-9: 橡皮擦功能完整
- **Given**: 用户使用橡皮擦
- **When**: 擦除圈画
- **Then**: 可以调节橡皮大小和形状
- **Verification**: `human-judgment`
