# 单词段落生成工具 - 产品需求文档 (PRD)

## 1. 产品概述

一款面向考研英语学习的纯前端网页应用，对接 DeepSeek API，实现百词斩每日单词批量导入、AI批量生成连贯阅读段落、单词释义交互查看、数据本地分类归档等功能。

- **目标用户**：考研英语备考学生
- **核心价值**：通过AI生成的连贯阅读段落帮助用户在真实语境中记忆单词，提升单词记忆效率和阅读理解能力
- **使用场景**：每日背单词后的复习巩固，配合百词斩等背词工具使用

## 2. 核心功能

### 2.1 用户角色

| 角色 | 使用方式 | 核心权限 |
|------|----------|----------|
| 单用户 | 本地打开index.html | 全部功能可用 |

### 2.2 功能模块

1. **主页面**：单词导入、词单管理、AI文本生成、文本展示、历史记录、配置管理

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主页面 | API配置面板 | 输入DeepSeek API地址、密钥、超时时间；测试连接；配置持久化 |
| 主页面 | 全局样式配置 | 亮色/暗色主题切换；单词高亮样式自定义（加粗、底色标黄、字体变色） |
| 主页面 | 文件保存配置 | 自定义保存路径；自动命名规则：年月日_复习词+新词_版本号.txt |
| 主页面 | 单词导入区 | 批量粘贴导入（空格/换行分割）；手动单条补词；自动去重 |
| 主页面 | 词单管理区 | 复习词/新词分组管理；单词列表展示（英文+释义+词性）；状态标记（已掌握/待复习/难点词）；删除/修改单词 |

| 主页面 | 文件归档区 | 保存归档按钮；导出TXT文件 |

## 4. 用户界面设计

### 4.1 设计风格

- **主题色**：亮色模式以白色为主背景，暗色模式以深灰/深蓝为主背景
- **强调色**：考研相关元素使用暖色调（橙色/金色），单词高亮使用黄色底色或蓝色字体
- **按钮样式**：圆角矩形按钮，适中尺寸适配手机触屏，hover状态有轻微阴影变化
- **字体**：中文使用思源黑体/微软雅黑，英文使用简洁无衬线字体，字号适中护眼
- **布局风格**：三栏布局（电脑端），左侧历史记录+词单，中间文本展示，右侧配置区；手机端自动转为上下分区
- **图标风格**：极简线性图标，无第三方图标库，使用CSS原生绘制

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 主页面 | 左侧区域 | 历史记录日期列表（卡片式）；词单列表（列表式）；分组切换按钮（Tab式）；状态筛选按钮（标签式） |
| 主页面 | 中间区域 | 功能按钮组（顶部固定）；文本展示区（多段落分页）；生僻词汇总栏（底部固定）；单词高亮样式（用户自定义） |
| 主页面 | 右侧区域 | API配置面板（折叠式）；主题切换（开关式）；高亮样式选择（下拉/按钮组）；保存路径设置（输入框）；全局按钮（底部） |
| 主页面 | 单词卡片 | 简易卡片：内嵌展开，词性+释义，三点按钮；详情弹窗：居中弹出，音标+全量释义+拓展，关闭按钮 |
| 主页面 | 弹窗提示 | 成功/失败提示弹窗，简约风格，自动消失或手动关闭 |

### 4.3 响应式设计

- **设计原则**：桌面优先，移动端自适应
- **电脑端**：三栏并排布局，左侧固定宽度280px，右侧固定宽度240px，中间自适应
- **手机端**：单栏布局，左侧区域移至顶部（可折叠），中间区域为主，右侧配置区移至底部（可折叠）
- **触屏优化**：按钮点击区域最小44px，列表项点击区域适中，避免误触

## 5. 数据存储设计

### 5.1 本地存储结构

| 存储键 | 存储内容 | 说明 |
|--------|----------|------|
| `vocab_config` | API配置、主题设置、高亮样式、保存路径 | 全局配置持久化 |
| `vocab_words` | 当日词单数据（复习词、新词、状态标记） | 词单数据 |
| `vocab_history` | 历史记录（日期、词单、文本版本、状态） | 历史数据归档 |
| `vocab_current_text` | 当前生成的文本内容、版本号 | 当前文本 |

### 5.2 数据格式示例

```json
{
  "vocab_words": {
    "date": "2026-06-14",
    "reviewWords": ["word1", "word2"],
    "newWords": ["word3", "word4"],
    "wordDetails": {
      "word1": {
        "translation": "释义",
        "pos": "n.",
        "status": "待复习"
      }
    }
  }
}
```

## 6. 异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| API网络失败 | 弹窗提示"网络连接失败，请检查网络" |
| API密钥错误 | 弹窗提示"API密钥无效，请检查配置" |
| API返回空内容 | 弹窗提示"生成失败，请重试" |
| 导入空内容 | 弹窗提示"未检测到有效单词" |
| 空词单生成拦截 | 弹窗提示"词单为空，请先导入单词" |
| 重复单词导入 | 弹窗提示"已过滤X个重复单词" |

## 7. 技术约束

- 纯前端实现，无后端服务
- 使用原生HTML5 + CSS3 + JavaScript
- 不使用Vue/React等框架
- 使用localStorage存储数据
- 使用fetch调用DeepSeek API
- 响应式布局适配电脑和手机
- 所有Prompt内置在代码中，用户不可修改
